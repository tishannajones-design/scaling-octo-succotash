A comprehensive design language for building information-enabled governance interfaces. Version 1.0 — Publish Ready.

Color system with 10 brand colors, 50+ semantic variants, dark mode support, and WCAG AA/AAA contrast compliance. 9-level type scale with responsive breakpoints. 8px spatial grid with 12-column layout system.

Brand Foundation

This system implements the RGX Engage visual identity: Strategic Navy and Flow Blue as primary brand expressions, Montserrat for display typography, Inter for body text, and a dual-service color architecture differentiating GaaS (Blue + Orange) from GPaaS (Purple + Green).

Design Principles
Five guiding principles that shape every design decision across the RGX Engage ecosystem.
01
Information Clarity

Every element must optimize information flow. Governance interfaces surface the right data at the right time—never overwhelming, always actionable. Visual hierarchy serves decision-making.

02
Context Sensitivity

Components adapt to organizational maturity. A startup's dashboard differs from an enterprise's compliance view. The system provides variants that match the user's governance context.

03
Execution Focus

Every interaction leads toward action. Buttons have clear purpose, forms minimize friction, and feedback is immediate. The interface drives governance execution, not just observation.

04
Human-Centered Governance

Governance frameworks work when people adopt them. Every component prioritizes usability, clear language, and progressive disclosure. Complex governance becomes approachable through thoughtful design.

05
Systematic Consistency

Every pixel follows the 8px grid. Every color meets contrast ratios. Every component uses the same interaction patterns. Consistency builds trust—and trust is the foundation of governance. The system scales from a single assessment to an enterprise-wide platform without losing coherence.

✓ Do

Use exact hex values from the palette. Maintain GaaS/GPaaS service differentiation. Use Strategic Navy for primary backgrounds. Limit to 2–3 colors per composition. Test all combinations for WCAG AA contrast. Use the 60-30-10 rule: 60% neutrals, 30% brand, 10% accent.

✗ Don't

Mix GaaS colors (Blue+Orange) with GPaaS colors (Purple+Green). Use colors outside the defined palette. Use pure black (#000000)—use Context Grey instead. Place Yellow or Red as full backgrounds. Use Orange or Green for body text (insufficient contrast). Rely on color alone to convey information.

Accessibility Notes

Minimum body size: 16px (1rem) ensures readability for all users. Never set body text below 14px.

Line height: Body text uses 1.5× for optimal readability. Headings use tighter leading (1.1–1.25×) to maintain visual cohesion.

Line length: Aim for 60–75 characters per line (max-width: 640px at 16px body). Exceeding 80 characters degrades readability.

Letter spacing: Display text uses -0.02em for visual tightness. Caption text uses +0.01em for legibility at small sizes.

Responsive: Display and H1–H3 scale down at tablet (≤1024px) and mobile (≤640px) breakpoints. Body and below remain constant.

Form Guidelines

Label placement: Top-aligned labels for scannability. Required fields marked with * asterisk.

Field spacing: 20px (sp-5) between field groups. 4px between label and input.

Accessibility
WCAG 2.1 AA compliance guidelines for all components. These standards are non-negotiable across the RGX Engage ecosystem.
Color & Contrast

Normal text: minimum 4.5:1 contrast ratio.

Large text (18px+ or 14px bold): minimum 3:1.

UI components and graphical objects: minimum 3:1.

Never rely on color alone to convey information — pair with icons, text, or patterns.

Test with deuteranopia, protanopia, and tritanopia simulators.

Keyboard Navigation

All interactive elements must be reachable via Tab key.

Focus indicator: 2px solid blue outline at 2px offset.

Escape closes modals, dropdowns, and tooltips.

Arrow keys navigate within composite widgets (tabs, menus, radios).

Enter/Space activates focused elements.

ARIA Patterns

Buttons: native <button> elements. No div with role="button".

Modals: role="dialog" aria-modal="true" with focus trap.

Alerts: role="alert" for inline, role="status" aria-live="polite" for toast.

Forms: every input has a visible <label> with for attribute. Error messages linked via aria-describedby.

Tables: use <th scope="col"> headers. Complex tables get aria-label.

Motion & Animation

Respect prefers-reduced-motion: disable transitions and animations.

No auto-playing animations longer than 5 seconds.

Provide pause/stop controls for any persistent motion.

Skeleton loaders and spinners are exempt but should be subtle.

Developer Guide
Implementation reference for engineers building with the RGX Engage design system.
Getting Started

The RGX Engage design system is built on CSS custom properties (design tokens), semantic HTML, and accessibility-first patterns. All tokens are defined on :root and can be consumed in any CSS-in-JS or plain CSS project. 

/* 1. Import fonts */
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Montserrat:wght@500;600;700;800&family=JetBrains+Mono:wght@400;500&display=swap');

/* 2. Import design tokens (or paste :root block) */
@import url('./tokens.css');

/* 3. Apply base styles */
body {
  font-family: var(--font-body);
  font-size: var(--text-body);
  color: var(--navy);
  background: var(--white);
  line-height: 1.5;
  -webkit-font-smoothing: antialiased;
}
